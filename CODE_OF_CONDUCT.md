# Code of Conduct - KSA Open Data MCP

KSA Open Data MCP supports high-trust public-sector use cases. We expect respectful collaboration, professional communication, and a shared commitment to technical excellence.

## Commitment

This project is maintained for reliability and trust in public-sector contexts. Every participant is expected to help preserve that standard.

## Our Standards

We are committed to providing a welcoming and harassment-free environment for all contributors.

Positive behavior includes:

- using respectful, constructive language,
- accepting and giving actionable feedback,
- prioritizing project and user safety over ego,
- owning mistakes and improving quickly.

Unacceptable behavior includes:

- harassment, discrimination, or derogatory remarks,
- personal attacks, trolling, or deliberate disruption,
- publishing private information without consent,
- threats, intimidation, or coercive behavior.

## Enforcement

Maintainers are responsible for interpreting and enforcing this Code of Conduct fairly and consistently.

If you experience or witness unacceptable behavior:

- open an issue when appropriate, or
- contact maintainers privately for sensitive matters.

Maintainers may take actions including:

- warning,
- temporary suspension from project spaces,
- permanent removal from project spaces,
- PR rejection or reversion when necessary.

## Scope

This Code of Conduct applies across all official project spaces, including:

- repository issues and pull requests,
- discussions and community channels,
- project-operated communication contexts.

## Reporting Guidance

When reporting an incident, include:

- what happened,
- where and when it occurred,
- links or evidence if available,
- any immediate safety concerns.

Reports will be handled with discretion and good-faith review.

## Good-Faith Clause

Maintainers and contributors are expected to:

- assume positive intent while facts are being established,
- avoid escalation through public confrontation,
- prioritize evidence-based resolution and project safety.

Thank you for helping keep KSA Open Data MCP safe, constructive, and professional.
